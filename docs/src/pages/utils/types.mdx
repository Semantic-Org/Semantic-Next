The utility library provides various helpers for checking `types`

## Functions

### isObject

```typescript
function isObject(x: any): boolean
```

Checks if the given value is an object.

#### Parameters

| Name | Type | Description |
|------|------|-------------|
| x    | any  | Any value to check |

#### Returns

`boolean`

#### Example

```javascript
import { isObject } from '@semantic-ui/utils';

isObject({}); // true
isObject([]); // true
isObject(null); // false
```

### isPlainObject

function isPlainObject(x: any): boolean

### isClassInstance

```typescript
function isClassInstance(obj: any): boolean
```

Checks if the given object is an instance of a custom class (not a built-in object).

#### Parameters

| Name | Type | Description |
|------|------|-------------|
| obj  | any  | Any object to check |

#### Returns

`boolean`

#### Example

```javascript

class MyClass {}
isClassInstance(new MyClass()); // true
isClassInstance({}); // false
isClassInstance(new Date()); // false
```

## Constants

### isServer

A boolean constant indicating whether the current environment is server-side.

- **Type:** `boolean`

#### Example

```javascript
import { isServer } from '@semantic-ui/utils';

if (isServer) {
  // Perform server-side specific operations
}
```

### isClient

A boolean constant indicating whether the current environment is client-side (browser).

- **Type:** `boolean`

#### Example

```javascript
import { isClient } from '@semantic-ui/utils';

if (isClient) {
  // Perform client-side specific operations
}
```

These type checking utilities provide a robust set of tools for validating and working with different data types in JavaScript, enhancing code reliability and reducing type-related errors.
