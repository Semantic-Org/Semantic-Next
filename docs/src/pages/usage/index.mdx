## Adding Components

You can use web components just like normal tags like `<p>` `<h3>` etc in your page. Simply add the tag wherever you want the component to appear in your `html`.

Web components can be quickly identified because they are [required by spec](/) to have a `-` dash in their names so `<ui-button>` but not `<button>`.

## Attributes

Semantic is built to support **three dialects**, ways which attributes are passed into components, that can be mixed/matched to allow you and your team to decide how to structure your code.

* **Standard** - Standard lets you use words and have the component interpret their classification. This is the most similar to natural language.
* **Technical** - A more explicit syntax where attributes are enumerated by category.
* **Classic** - This syntax can be more familiar for seasoned developers, and lets you use class names to reference elements in code.

## Plurality

Some components support plurality, attributes assigned to the group pass through to the children. For instance in this example a button group is large so all child buttons are `large`, with one also being `primary`.


<UIButtons large>
  <UIButton primary>Large Primary</UIButton>
  <UIButton>Large</UIButton>
  <UIButton>Large</UIButton>
</UIButtons>
